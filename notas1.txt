## Existing architecture

Stack: PHP 8.3, Nginx, PostgreSQL 16, Docker Compose. Custom MVC — no framework.

Relevant structure:
- `src/Controllers/` — controllers extending `App\Core\Controller` (which is abstract and currently empty)
- `src/Services/` — service classes instantiated directly inside controllers
- `src/Views/` — PHP view files rendered via `\App\Core\View::render('ViewName')`
- `src/routes.php` — where all routes are registered on a `$router` instance
- `.env` / `.env.example` — environment variables loaded via vlucas/phpdotenv

Route registration syntax:
```php
$router->add('GET',  '/path',        'SomeController@method');
$router->add('POST', '/api/path',    'SomeController@method');
```

The Router instantiates controllers with `new ClassName()` and calls the method directly. Route params like `{id}` are passed as method arguments.

## What I need you to implement

### 1. Add helper methods to `src/Core/Controller.php`

Add the following protected methods to the base `Controller` class (matching the pattern already used by `DietHelperController`):

- `requireJson(): void` — checks that `Content-Type: application/json` is present; responds 415 and exits if not.
- `parseBody(): array` — reads and JSON-decodes `php://input`; returns empty array on failure.
- `json(array $data, int $status = 200): void` — sets `Content-Type: application/json`, sets HTTP status, echoes `json_encode($data)`, and exits.

### 2. Add API methods to `src/Controllers/KitchenHelperController.php`

Add the following methods to the existing class (keep `index()` untouched):

**`single(): void`**
- Accepts `POST` with JSON body: `{ "ingredients": ["chicken", "rice"], "sort": "healthiness"|"time"|null }`
- Validates that `ingredients` is a non-empty array; responds `['error' => 'Ingredients required']` with HTTP 400 if invalid.
- If `sort` is `"healthiness"` or `"time"`: calls `$service->searchRecipes()` passing `['includeIngredients' => implode(',', $ingredients), 'sort' => $sort, 'sortDirection' => 'desc']`.
- If `sort` is absent or null: calls `$service->searchByIngredients($ingredients, 12, true)`.
- Responds `['success' => true, 'data' => $results]` on success.
- Catches `RuntimeException` and responds `['error' => $e->getMessage()]` with HTTP 502.

**`mealPrep(): void`**
- Accepts `POST` with JSON body: `{ "ingredients": ["chicken", "rice"], "count": 3 }`
- Validates non-empty `ingredients`; responds 400 if invalid.
- Clamps `count` to the range 2–5.
- Calls `$service->searchByIngredients($ingredients, $count * 5, true)` to fetch a wider pool.
- Selection algorithm: sort the results by `usedIngredientCount` descending, then `missedIngredientCount` ascending. Take the first `$count` recipes.
- Responds `['success' => true, 'data' => $selected]` on success.
- Catches `RuntimeException` and responds `['error' => $e->getMessage()]` with HTTP 502.

**`recipeDetail(int $id): void`**
- Accepts `GET` with route param `{id}`.
- Calls `$service->getRecipeInfo($id, true)`.
- Responds `['success' => true, 'data' => $result]` on success.
- Catches `RuntimeException` and responds `['error' => $e->getMessage()]` with HTTP 502.
